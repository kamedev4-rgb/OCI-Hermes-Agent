# コーディングタスク委譲ガイド

## このスキルを使う場面

以下のすべてに当てはまる場合に delegate_task で GLM-5.1 に委譲する:

- ソースコードファイル（.py / .ts / .js / .go / .sh 等）の作成または修正が必要
- 変更対象のファイルパスが確定している
- テスト確認コマンドが存在する

## 委譲してはいけない場合

- .md / .yaml / .toml / .env / .json（設定ファイル）の変更
- git commit / git push の実行
- 新しいサービスの設計判断が必要な場合

## 委譲時のテンプレート

delegate_task を呼ぶとき、context には必ず以下を含める:

goal: "（変更内容を1文で）"
context: |
  プロジェクト: /home/ubuntu/（プロジェクトディレクトリの絶対パス）
  対象ファイル: （変更するファイルの絶対パスを列挙）
  禁止事項:
    - .md ファイルの作成・変更
    - .yaml / .env / .toml ファイルの変更
    - git commit / git push の実行
  テスト確認: （テストコマンドを記載）
  完了条件: テストが合格すること
toolsets: ["terminal", "file"]

## 委譲後の確認

GLM からの結果を受け取ったら、以下を自分で確認してから kame-dev に報告する:

1. テストコマンドを実行して合格しているか
2. 変更されたファイルが .py/.ts 等のソースコードのみか
3. git status で意図しない変更がないか
